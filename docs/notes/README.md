# Notes

This directory is the low-friction home for topical notes.

Use it for:

- investigation notes
- troubleshooting notes
- migration scratchpads
- one-off findings
- links and references
- operational reminders

These notes do not need the structure or review burden of the curated docs in
`docs/`.

## Guidance

- prefer writing a rough note over keeping knowledge in your head
- if a note becomes durable and broadly important, promote it into a curated doc
- avoid over-editing notes; optimize for capture first

## Suggested Naming

- `yyyy-mm-dd-topic.md`
- `area-topic.md`
- `incident-topic.md`
